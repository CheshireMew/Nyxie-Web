# 真实浏览器验收

用于证明角色网站真的启动、媒体真的传输、导演真的消费、用户真的看到结果。函数单测只能补充，不能代替本清单。

## 1. 验收前准备

确认：

- 使用的是最终网页媒体，不是原始生成文件或占位视频。
- 本地服务和生产环境采用与实际部署一致的媒体路径。
- 浏览器控制台、网络面板和媒体元素状态可观察。
- 测试会执行真实点击、滚轮、触摸或键盘操作。
- 当前工作区的用户改动已识别，不因验收覆盖或删除。
- 用户已经明确接受、暂时忽略或要求之后再处理的问题已单独记录；这些问题不重复判为阻塞，但仍检查本轮是否引入新问题。
- 滚动和媒体实现开始前已经写好性能职责、互动路由和最终稳定状态断言；已有项目修复时，至少在修改代码前还原这些合同。验收执行同一组条件，不根据当前实现反向降低标准。
- 每份旧版本、截图、录屏、测试和部署地址都标明时间、分支或来源及可信度。用户说明 Git 过时时，不把 Git 快照当作重构前视觉真源，改用用户最新决定、当前可观察结果和逐项确认的保护合同。

记录环境：

```text
地址：
浏览器与版本：
视口：
网络条件：
减少动态效果：开 / 关
声音：开 / 关
测试时间：
```

如果是已有网站迁移或局部改版，同时记录：

```text
要求保持不变的页面和状态：
加载态基准截图：
桌面首屏基准截图与视口：
移动首屏基准截图与视口：
关键互动基准录屏或说明：
允许重新设计的区域：
基线来源、日期与可信度：
```

先检查保护区，再检查新功能。React、Vue、GSAP 或其它实现改变不能作为视觉变化的理由。

## 2. 素材生产边界

对媒体清单中的每个文件检查：

- 文件存在且非零。
- 浏览器支持编码格式。
- 分辨率、帧率和像素格式一致或有明确原因。
- 实际时长与媒体清单一致。
- 裁剪后的动作高潮和片尾都可用。
- 锚点图尺寸和页面比例正确。

可用 `ffprobe` 获取确定性信息，但最终仍需播放检查画面。

## 3. 传输边界

启动网站后检查网络请求：

- 锚点图返回成功并在首个视频前可见。
- 首个待机视频返回 200 或按 Range 返回 206。
- 服务器按实际文件提供正确的媒体类型，例如 MP4 为 `video/mp4`，WebM 为 `video/webm`。
- Range 响应包含合理的 `Accept-Ranges` 和 `Content-Range`。
- 后续视频只按导演计划预加载，没有加载同一素材的无限请求。
- 需要提前到达的 Gallery 媒体由生产代码在合同触发点发起；请求 URL、会话选中形态与最终消费者一致，不是测试脚本补发的请求。
- 文件 404 或解码失败时，启动屏不会永久遮挡，锚点和导航仍可用。

## 4. 首次加载

步骤：

1. 在导航前安装最早期观察或录屏，必要时延迟应用入口脚本，捕获服务器 HTML 刚绘制、应用接管、加载界面和首页稳定四个时点；不能等应用挂载后才开始截图。
2. 以新页面打开网站，不先点击。
3. 在启动界面消失前截取加载态，检查标志、文案、进度、背景和退场时机。
4. 等启动界面结束。
5. 读取当前视频的 `src`、`readyState`、`paused`、`currentTime` 和 `duration`。
6. 观察几秒，确认 `currentTime` 持续增长。

通过标准：

- HTML 首帧显示合同允许的静态加载壳，应用接管后保持同一加载体验，随后进入首页；正常启动过程不闪现只供无 JavaScript、入口加载失败或明确超时使用的故障降级页。
- 锚点先提供完整首屏，不出现空白背景。
- 首个待机实际 `play()`，不是只设置了文件名。
- 导航没有遮挡角色头部或关键特征。
- 文案和按钮可读，不随视频模糊。
- 被标记为保留的加载动画和首屏，在相同视口、相同媒体状态下与基准的构图、裁切、遮罩、标题、控件和文案一致。

加载态、HTML 首帧、应用接管、转场峰值、视频结束态等瞬时界面必须在真实时机观察。只在页面稳定后读取 DOM，不能证明这些状态没有被改坏。故障降级页若属于合同，另用真实禁用脚本、入口加载失败或超时路径检查；不得为了“测试方便”让它进入正常成功链路。

## 5. 自主表演

步骤：

1. 不操作页面，观察至少一轮计划序列。
2. 记录每次片段键名、开始时间和去向。
3. 在切换点观察人物、锚点和媒体舞台。

通过标准：

- 顺序符合体验合同。
- 随机动作不会连续重复。
- 备用播放器在切换前已经可用。
- 锚点没有清晰停留成“图片—视频—图片”的卡顿循环。
- 没有黑帧、双人重影或转场外露出的硬切。
- 切换结束后媒体舞台恢复清晰，导演的当前键名与可见视频一致。
- 新片段就绪前只在隐藏备用层存在；晋升活动身份与可见层在同一稳定提交中完成，任一稳定帧最多一个人物层可见。旧异步操作结束时不会暂停、归零、显示或重播已经复用为当前人物的节点。

## 6. 偶然点击与明确操作

分别执行：

1. 点击角色画面。
2. 点击真正空白区域。
3. 点击导航、标题文案、链接、动画按钮和随机按钮。
4. 在弹窗打开时点击弹窗内外。
5. 在可见链接卡片内部、卡片外但仍位于同一作品区的位置分别发送真实坐标点击，记录 `elementFromPoint()`、当前 URL 和新页面 target 数量。
6. 打开弹层后，在弹层外、底层作品区域内点击同一坐标，确认命中的是遮罩而不是底层链接；再点击弹层内真实入口。

通过标准：

- 角色和空白区域触发约定的直接回应。
- 导航、文案和按钮不会附带触发空白点击回应。
- 一个点击只产生一个最高优先级请求。
- 明确按钮播放指定动作或执行未来替换后的功能。
- 叙事锁定期间其它操作不会抢走视频。
- 返回欢迎或自动动作正在转场时点击明确按钮，用户最后一次有效操作会在当前转场后执行，不会被静默丢弃。
- 链接只在可见入口范围内工作；卡片外点击不打开新页面，非入口作品展示本身不是大面积 `<a>`。
- 弹层遮罩真实拦截底层，弹层外点击不会穿透；弹层内入口仍能正常打开约定地址。

## 7. 第一次滚动剧情

这是关键测试，不使用消费端手写 `ended = true` 或直接调用滚页函数。

步骤：

1. 从下方章节执行一次真实整页刷新；如果合同要求刷新回到首页，确认导航类型为 `reload`、`history.scrollRestoration` 和最终 `scrollY`，不能先用测试代码滚回顶部。
2. 发送一次真实向下滚轮。
3. 在导演等待、媒体转场和播放三个阶段分别尝试空白点击、动作按钮、随机播放、导航、键盘和连续滚轮。
4. 记录剧情片段的 `duration`、开始时间、播放速度、静音状态和独占锁。
5. 如果合同规定用开门过程预热 Gallery，从请求受理起记录目标形态 id、视频和海报请求开始、目标资源可用、剧情结束、Gallery 到达及人物可见的时间戳；到达前确认预热视频保持暂停。
6. 在片尾读取 `currentTime`、`ended` 和页面 `scrollY`。
7. 等浏览器实际触发 `ended`，观察页面何时滚动。
8. 等滚动稳定，再读取目标章节的 `getBoundingClientRect().top`、根元素 `scroll-padding-top`、活动导航和用户实际可见内容。
9. 通过站内导航回到首页顶部，再发送真实向下滚轮；确认页面原生移动而不重播剧情。
10. 再执行一次整页刷新并发送向下滚轮；如果合同规定每次完整加载重置，确认剧情重新可用。

通过标准：

- 第一次有效滚轮启动剧情片段。
- 请求受理后立即建立独占锁；等待导演、切换播放器、播放和导航稳定期间，所有竞争入口都不能改变片段或目标章节。
- 播放期间连续滚轮均被锁住，首屏没有逐渐露出下一章节。
- 页面开始滚动时，目标播放器满足 `ended === true`，且 `currentTime` 接近真实 `duration`。
- 导演没有在片尾前切回待机。
- 成功播放、同一入口的无动画备用路径、滚动提示按钮和减少动态效果路径使用同一互动路由目标；“下一章节”来自有序章节真源，不在各事件监听器里分别硬编码。播放中途的意外失败仍按失败合同停留并提供重试。
- 最终稳定状态中，目标章节达到互动合同约定的导航锚点；默认顶部对齐时，其顶部与固定导航留白一致。活动导航、目标内容和互动合同必须一致；平滑滚动途中短暂经过目标章节不算通过。
- 动画失败或超时时停留在首屏、恢复输入并提供重试，不假装成功滚页。
- 回到顶部不重播属于同一文档状态；整页刷新才按合同重新创建剧情状态。滚动位置、组件重新入场和 `ended` 都不能自行重置单次消费标记。
- 合同要求预热时，Gallery 的目标请求在用户到达前已由生产代码发出并达到约定可用状态；到达后立即显示同一形态，激活时从合同起点播放。等待条件同时绑定预期形态 id、具体节点和 `currentSrc`，不能由任意活动视频、旧节点的 `ready` 事件或空资源列表提前满足。

触摸端重复测试：从首屏向上滑超过阈值，播放期间继续滑动，结果应与滚轮一致。

## 8. 离开和返回

步骤：

1. 正常进入下一章节。
2. 确认首屏视频暂停。
3. 通过导航返回首屏。
4. 再通过下方动作入口返回首屏并要求播放指定动作。

通过标准：

- 离开时不会继续无意义解码和自动切片。
- 普通返回只触发一次欢迎回应。
- 指定动作返回时不会同时触发欢迎回应。
- 欢迎回应、普通待机或指定动作都不会重置首屏单次剧情；返回顶部后的下一次向下输入按合同原生滚动。
- 返回后的可见视频、导演键名、按钮状态和进度一致。

## 9. 声音、页面可见性与错误恢复

测试：

- 从正式媒体清单遍历每个 Hero 片段、首屏剧情、每个 Gallery 形态、Personality 和独立 BGM；让生产入口实际选择每一项，并在初次播放、播放器交换、重播或失败恢复后读取真实播放器的 `playbackRate`、`defaultPlaybackRate`、`muted`、`preload` 与完成策略。不要只抽查当前视频，也不要在测试里重复维护速度常量。
- 对 `loop`、`hold`、`replay` 和 `advance-on-ended` 分别按项目合同验证真实片尾结果。Gallery 使用 `advance-on-ended` 时，等待真实 `ended`，确认下一人物经过同一选择边界、目标媒体已就绪且全过程最多一个人物可见；没有该策略时不得被遗留监听器自动切换或重播。
- 人物视频按合同保持静音并能自动播放。声音按钮存在时，确认它控制的具体声音通道，而不是默认修改所有 `<video>`。
- BGM 默认关闭时保持暂停；真实点击开启后从约定起点播放，接近片尾后实际循环，关闭后暂停，标签与 `aria-pressed` 同步。
- 如果采用 Blob 避免下载器接管，确认原始音频由生产代码真实请求成功，`<audio>` 的 `src` / `currentSrc` 只有内部 `blob:`，真实时长和约定区间一致；不能由测试脚本手工塞入 Blob。
- 页面切到后台，当前视频暂停；回到前台按当前状态恢复。
- 在切换时模拟一个真实视频 404 或不可解码文件。

通过标准：

- 自动播放策略失败不会造成永久停住。
- 声音按钮不会解除合同规定为始终静音的人物视频；BGM 的默认值、循环、音量、真实时长和用户最新决定一致。
- 页面媒体元素没有暴露会被下载器当作直接下载任务的音频文件 URL，同时真实音频仍由页面读取并被播放器消费。
- 新视频失败时恢复旧视频或锚点，媒体舞台不永久模糊。
- 导演解除切换锁，可以继续接受有效请求。
- 控制台报告可理解的错误，用户仍可浏览页面。
- 每一项的实际速度、静音与完成结果都来自当前合同；用户修改其中一项后，代码、媒体清单、测试、文档和维护样本不存在仍断言旧值或旧 `loop` 行为的路径。

## 10. 响应式和减少动态效果

至少检查：

- 常见桌面视口。
- 约 390×844 的移动视口。
- 粗指针或真实触摸模式。
- `prefers-reduced-motion: reduce`。

通过标准：

- 角色脸、头部和关键动作没有被导航或裁切破坏。
- 文字和控制不会盖住角色主要表演区域。
- 移动端无需悬停即可访问主要功能。
- 减少动态效果模式显示静态锚点，导航和章节仍可使用。
- 已有移动端控制、文案和互动不能因为“简化移动布局”被擅自隐藏；只有变更合同允许时才调整。

## 11. 后续章节的生命周期、信息和滚动成本

逐章使用真实滚轮或触摸走完整链路。先写下章节类型、主架构、信息任务、主要证据和反向策略，再在桌面与移动端记录 `section.offsetHeight / window.innerHeight`。每个有章内演出的章节固定取证五次：

| 检查点 | 必须读取的生产结果 | 通过标准 |
| --- | --- | --- |
| 进入构图 | 章节与舞台的 `getBoundingClientRect()`、章内真实进度、可见状态 | 章节尚未完整到锚点时，章内进度严格为 0，后段编号、文案或人物状态尚未出现 |
| 完整就位 | 舞台顶部与导航留白、人物头部、标题和关键编号几何 | 舞台完整、无遮挡，章内进度仍为 0，下一页不在视口 |
| 演出中段 | 真实滚动位置、章内进度、当前离散状态、下一页顶部 | 舞台保持完整，进度位于 0 与 1 之间，只有合同对应状态可见，下一页顶部不小于表演视口底部 |
| 终态释放 | 章内进度、最终状态、下一页首次可见位置 | 下一页首次进入视口之前，本章进度已经为 1 且所有必须信息稳定可见 |
| 反向返回 | 第一次有效反向输入前后的滚动位置、阶段和可见状态 | 普通章节不倒放、不重新入场；第一下就产生位移或状态反馈，不经过长距离空滚 |

验收脚本可以读取页面由生产代码暴露的阶段、进度和当前形态，但不得直接设置 dataset、时间线进度、媒体 ended 状态或滚动完成标志。关键截图必须由真实输入经过生产者、章节导演和浏览器布局后得到。

按主架构补充检查：

- 原生滚动 + sticky 允许正常页面交界，但完整就位前不能开始章内演出，演出期间不能露出下一页。
- 固定全屏舞台换场在整个过渡中只显示一个完整舞台，不能出现上下两页拼接；导航和可访问内容仍与当前状态一致。
- 分页吸附只有在体验合同明确接受时才通过，并用鼠标滚轮、触控板、触摸和反向返回分别检查是否需要重复输入。
- 被动内容不伪造固定距离，也不为了统一效果强行进入章内时间线。

Gallery 另走显式局部交互链路：先从生产代码读取完整目录、合同规定数量的随机样本、当前形态和已看样本，确认随机样本在本次完整页面加载中稳定且只是完整目录的子集。完整就位前滚轮不切人物；向下每个有效手势只推进一个未看样本；样本完成后下一次向下自然离开；向上第一下立即返回上一章，不倒序消费样本。再用左右按钮、方向键和侧卡遍历完整目录，至少实际加载一个随机样本之外的形态，确认各入口共用同一选择、转场、播放速度、静音与完成策略边界；返回任意形态时按合同从头播放或恢复。从下方向上重新进入 Gallery 后，当前活动视频应继续或恢复播放，不能仅因滚动方向为负而静止。每次切换按预期形态 id、具体节点和 `currentSrc` 等待就绪，并逐帧或录屏检查可见人物层数量；“最后看到正确人物”不能排除中间曾重叠或被旧清理回调重播。

末章弹层、链接卡片或其它“滚到底自动出现”内容，以章节稳定终态为生产条件。在视口左、中、右三个横向位置分别发送相同向下滚轮，最终都应在同一章节进度出现；如果只有鼠标位于中间才触发，先检查输入监听挂载范围、覆盖层和 hover 条件，不能用提前阈值掩盖错误所有权。

内容与设计同时通过：

- 除非合同明确声明有意停留，否则不应连续滚动约一屏仍没有新的信息、构图关系或状态。
- 任意抽样进度都有清楚焦点，没有空白失焦、重复状态或用漂浮素材填时间的段落。
- 角色或风格 DNA 延续到后续章节，但没有遮住真实内容证据。
- 作品章节的真实项目名称、说明、实现亮点或结果证据和有效入口可以直接看到并使用。
- 独立装饰与相邻内容存在可见关系，移除会造成明确的信息、结构或交互损失；否则不应出现在 DOM、素材清单和动画调用点。
- 减少动态效果模式不保留没有信息价值的固定距离，静态内容顺序仍完整可读。

只验证选择器存在、图片成功加载、动画函数被调用或脚本能写入期望状态，不能证明用户得到这些结果。

## 12. 滚动性能专项

网站包含滚动驱动、固定舞台、Canvas 或多媒体章节时，执行编码前性能合同对应的浏览器检查。用户已经反馈卡顿、掉帧或动画不跟手时，完整读取 `references/scroll-performance.md`，并在真实浏览器中重复同一条滚动路径；至少保留一份修改前证据和一份修改后证据。如果修改前环境已经不可复现，必须明确写为未验证，不能补造基线。

步骤：

1. 固定视口、缓存状态、减少动态效果设置、起止章节和输入方式。
2. 记录主线程脚本、组件更新、样式与布局、绘制与光栅、合成、媒体掉帧和首次资源解码峰值。
3. 逐次只关闭或替换一个候选边界：每帧状态更新、高成本滤镜或 Canvas、视频层、首次资源预热。
4. 根据证据修正唯一生产者或消费边界，移除被替代的旧监听器、旧 action、旧动画调用和旧恢复逻辑。
5. 在冷启动、热缓存、桌面、移动端和减少动态效果模式复测真实滚动，并重新走开门动画和页面最终落点。

通过标准：

- 能说明主要成本位于输入、状态、布局、绘制、合成、资源解码或媒体解码中的哪一层，并有同一路径的对照证据。
- 实际代码的状态生产者、连续进度所有者、活动媒体和预热时序与编码前合同一致；出现偏差时修实现或重新取得用户确认，不能事后改合同掩盖偏差。
- 离散章节状态不会因每个滚动帧无变化地更新全局应用状态；连续视觉进度只有一个生产者。
- 合同触发点之后的资源预热确实由生产代码发起，真实图片或媒体被最终消费者读取；不能由测试在消费端手写 `ready` 状态冒充生产结果。用时间线证明请求与可用发生在章节到达前，并绑定当前会话目标身份。
- 修复没有用全局滚轮倍率、长期输入拦截、强制翻页或删除有信息价值的状态掩盖问题。
- 视频只有在真实 dropped frames、静态海报对照和解码条件共同指向它时才被判为主因。
- 用户可观察到的章节推进、媒体演出、导航和最终落点没有因性能修改而退化。

弹层、菜单和其它过渡界面也按最终稳定状态验收。原生 `dialog.open === true` 只能证明容器已经打开；如果合同要求面板进入、焦点落入和控件可用，还要等待视觉状态、最终几何位置和焦点同时成立。

## 13. 透明素材、内容和外部入口

透明图片和视频需要四层证据：

1. 生产者层：确认使用的是原始透明 MOV 或其它高质量 Alpha 母版，而不是已经有损压缩、只靠文件名宣称透明的中间文件。
2. 解码层：使用正确解码器读取母版和网页输出的 Alpha 平面，确认同时存在透明和不透明像素，并记录帧数、时长、全画面、外圈与四角分布。
3. 消费者层：网页视频和减少动态效果使用的海报来自同一份清理后输出，媒体清单、实际 `src` 和 poster 没有继续指向旧文件。
4. 页面层：在目标浏览器和真实章节背景上播放或显示，没有黑底、绿边、错误预乘、明显溢色或整幅矩形边界。

文件扩展名、编码标签、查看器背景颜色和单张封面都不能单独作为透明通过证据。不支持当前透明编码的浏览器必须显示约定的海报或其它回退。

### 低 Alpha 全画布方框

当浅色背景上能看到与视频画布同尺寸的淡框时：

1. 对高质量母版和网页输出分别解码真实 Alpha。VP9 WebM 要明确使用能读取 Alpha 的 `libvpx-vp9`，不能因为默认解码得到 `yuv420p` 就判为不透明或正常。
2. 统计 `0 < alpha <= threshold` 的全画面、外圈和四角占比。全画面数字用于发现问题，外圈和四角用于区分整幅底噪与人物边缘。
3. 从母版运行 `scripts/clean_alpha_video.py`，在 VP9 有损编码前清除低 Alpha，再生成 WebM 和海报；不从旧 WebM 二次压缩。
4. 比较清理前后帧数、时长和 Alpha 分布。VP9 可能在真实主体边缘重新产生少量低 Alpha，不能要求输出机械归零。
5. 把开始、25%、50%、75% 和结束附近的帧合成到网站实际背景上，并在真实浏览器播放；检查人物细发、薄纱、翅膀、尾巴、细链和粒子没有被阈值吃掉。
6. 批量形态逐个检查；一个视频通过不能替代其它文件。

通过标准：四周不再形成连续矩形，清理前高覆盖的四角或外圈显著下降，帧数和时长保持一致，真实半透明细节仍在，视频与海报在桌面、移动端和减少动态效果模式都没有方框。阈值统计和 `alpha_mode` 只提供证据，不单独决定通过。

内容区同时检查：

- 项目、博客和社交入口对应真实地址。
- 尚未提供的地址明确显示“待接入”，不生成看似真实的假链接。
- 场景道具没有被误写成角色原设。

## 14. 视觉保护区对照

对每个标记为“保留”的状态，在相同视口和尽量相同媒体帧下对照：

| 状态 | 对照内容 |
| --- | --- |
| 加载态 | 标志、文案、进度线、背景、最短展示时间、退场 |
| 桌面首屏 | 角色裁切、导航、标题、遮罩、按钮、控制组、滚动提示 |
| 移动首屏 | 角色焦点、标题顺序、动作按钮、声音和目录入口 |
| 关键互动 | 触发方式、动画、结束去向和返回状态 |

功能测试全部通过但保护区截图明显改变时，验收仍然失败。先恢复保护区，再继续新页面工作；不能把新的审美判断当成默认授权。

布局与指针反馈另外取真实几何：

- 在首屏加载、向下滚动、返回首页和整页刷新后，固定导航下沿与首屏媒体上沿的差值都符合合同；预期贴合时应接近 0，不能只量导航自身高度。
- 用户要求隐藏页面滚动条时，读取 Firefox `scrollbar-width` 和 WebKit scrollbar 样式，并用滚轮、键盘和程序外真实触摸确认原生滚动仍工作。
- 自定义光标先移动到普通区域并等待稳定，再快速移动到可点击入口。记录红点和外圈圆心的最大距离、停止后的距离以及 hover 视觉层缩放；动态跟随需要大于 0 且不超过合同上限，停止后回到同心，缩放不能改变定位圆心。
- 弹层打开前后分别在卡片内、卡片外和遮罩区域读取 `elementFromPoint()` 并真实点击，确认视觉层级和命中层级一致。

## 15. 运行环境清理

记录本次验收启动的开发服务器、浏览器调试实例和端口。交付前：

1. 停止本次启动且不再需要的服务和调试进程。
2. 检查默认端口是否仍在监听。
3. 如果用户自己的服务原本就在运行，不擅自结束；说明复用了哪个进程。
4. 从干净状态实际运行一次用户启动脚本，确认不会因为验收进程占用端口而失败。
5. 不向用户声称“可以直接启动”，除非默认端口已经释放或明确说明已有同一网站正在运行。

## 16. 验收报告

按以下格式交付：

```text
通过：
- 用户可观察结果 + 实际证据
- 后续各章节的实际屏数和进入 / 完整就位 / 章内演出 / 终态释放 / 反向画面证据

失败：
- 复现步骤
- 实际结果
- 期望结果
- 根因所在边界：素材 / 传输 / 导演 / 事件 / 状态 / 布局 / 绘制 / 合成 / 解码 / 导航

已接受问题：
- 用户明确允许保留的素材瑕疵
- 本轮没有扩大该问题的证据

未验证：
- 未验证内容
- 原因
- 需要的条件

清理：
- 已停止的本次进程
- 保留的用户进程及原因
- 默认端口最终状态
```

不要只写“运行正常”“测试通过”。关键结论要带真实片段、媒体事件、滚动位置或浏览器状态证据。
