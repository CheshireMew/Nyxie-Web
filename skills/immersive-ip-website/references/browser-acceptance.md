# 真实浏览器验收

用于证明正式页面从真实素材和生产状态得到用户能看到、能操作的结果。函数单测和手写状态只能补充，不能替代这条链路。

## 1. 记录合同与环境

```text
页面目标：
必须完整展示的内容：
互动输入、取样和反馈位置：
目标地址：
浏览器与版本：
视口与设备：
冷缓存 / 热缓存：
减少动态效果：
完整加载、站内导航、前进后退和页面缓存恢复合同：
要求保留的页面或状态：
允许改变的区域：
本轮最后一次影响结果的修改：
```

至少覆盖用户实际截图或明确提出问题的精确视口，再覆盖超宽屏、常规桌面、窄屏、移动端和减少动态效果模式。固定视口名称不能代替用户目标尺寸。

先确认异常的控制者。页面 DOM、样式、媒体、favicon 和站点元数据属于网站验收；浏览器原生界面、扩展注入层和操作系统浮层不属于页面 DOM。外部界面问题通过对应浏览器或扩展设置验证，不为了隐藏它改写媒体语义、把视频伪装成其它节点或加入无法控制的 CSS 选择器。

证据只对生成时的文件、配置和媒体版本有效。页面、样式、媒体、清单、测试或构建输入在验收后又发生变化时，重新运行受影响的真实链路；不能把修改前的完整报告和修改后的局部观察拼成最终通过。

## 2. 证明素材生产结果

对正式媒体清单中的文件检查：

- 文件存在且非零；
- 实际分辨率、帧率、时长、像素格式、编码和音轨；
- poster 与视频主体位置和构图一致；
- 可用起止区间与清单一致；
- WebM、MP4 或其它回退由目标浏览器真实支持；
- 透明媒体的母版、网页输出和 poster 来自同一有效链路。
- 截图、poster 和抽查的视频帧没有烘焙已经退役的导航、文案、按钮或页面状态。

命令成功、扩展名正确或播放器能打开都不能单独证明页面可用。最终仍要在目标浏览器和真实背景中观察。

### 2.1 验证选定视觉和全局品牌资产

用户选定的标志、图标或视觉稿需要同时检查：

- 放大轮廓中的折点、圆弧、缺口、负空间、笔画层级和颜色位置；
- 页眉、按钮或卡片中的实际显示尺寸，确认没有粘连、过细、压扁或识别变化；
- 用户只要求局部调整时，其它已锁定关系与上一确认版本一致；
- 页眉、favicon、应用清单、加载界面、分享元数据和烘焙媒体是否都在消费者清单中；
- 能共用同一矢量源的消费者是否实际读取同一路径，派生版本是否能追溯到同一母版；
- favicon 等强缓存入口的最终 `href`、文件内容和常见标签尺寸是否已经更新，必要时重新打开标签或使用新指纹验证。

不要用“都包含相同字母”或“整体风格接近”代替几何一致。位图转矢量时，先放大对照关键轮廓，再回到真实页面尺寸截图。

## 3. 证明首次可见链路

从新页面导航开始录屏或捕获关键帧，观察：

```text
服务器 HTML 首帧
→ 应用接管
→ 页面 poster 或静态锚点
→ 最终播放器首个可解码画面
→ 稳定播放状态
```

通过标准：

- 媒体出现前页面已有符合构图合同的完整画面，不出现空白舞台；
- poster 保持到最终消费者的目标画面可用，再稳定退场；
- 首个播放器确实执行 `play()`，`currentTime` 持续增长；
- 导航、标题、互动反馈没有在切换中闪到错误位置；
- 被要求保留的加载态和首屏与相同视口基线一致。

只在页面稳定后读取 DOM，不能证明首帧和加载过渡没有出错。

自动化等待正式页面的可观察就绪条件，例如目标章节已经激活、目标媒体 ID 与 `currentSrc` 一致、动画完成状态已经由生产时间线提交。固定延时只能作为超时上限；等待若超时就报告未就绪，不能在延时结束后直接把当时画面当成成功终态。

## 4. 证明预加载复用最终消费者

从合同中的预加载触发点开始记录时间线：

```text
触发时间
→ 正式选择逻辑得到的目标 ID 与 URL
→ 正式页面发出请求
→ 目标消费者或可晋升备用消费者达到可解码状态
→ 用户到达页面
→ 同一目标成为可见媒体
```

需要同时核对：

- 目标 ID；
- 具体媒体节点；
- `src` 与 `currentSrc`；
- `preload`、`readyState` 和对应媒体事件；
- Resource Timing 或 Network 中的真实请求；
- 到达后的可见节点与当前选择状态。

临时预加载元素、测试脚本补发请求、旧节点事件、任意 `readyState > 0`、空任务列表或最终另一次重新请求都不能通过。冷缓存下重复一次，确认到达页面时 poster 已在，媒体不经过空白等待。

对于图片，`complete === true` 不能单独通过；同时核对目标 `src` 或 `currentSrc`、`naturalWidth > 0`、`naturalHeight > 0`，并在正式链路需要时等待 `decode()`。在内容首次可能进入视口、但活动章节仍可能是上一章的位置取证，确认最终节点已经挂源并解码。

对于视频，Resource Timing 中同一 URL 的条目数量不能单独证明播放器被重复创建或复用成功。Range 请求、缓存和浏览器实现会改变条目数量。节点复用通过真实播放器数量、节点身份、预热标记或正式状态延续、目标 `currentSrc` 和播放状态证明；网络重复问题另行结合请求发起者、Range 和传输量判断。

## 5. 验证响应式构图

对每个目标视口截图并读取真实几何：

| 检查项 | 通过标准 |
| --- | --- |
| 人物完整范围 | 合同要求的头部、手、衣摆、鞋或其它轮廓均可见 |
| 人物与环境关系 | 动作、光线和场景接触关系没有因裁切失效 |
| 视觉焦点 | 第一与第二焦点顺序清楚 |
| 页面空间 | 每个空区承担留白、信息、导航或互动职责 |
| 信息区域 | 文案和控件可读，不压住主要动作和身份细节 |
| 互动反馈 | 位置、大小和边界符合合同 |
| 导航边界 | 固定导航不遮挡主体，也不额外制造无职责间隙 |

不要只检查“视频没有溢出”。如果超宽屏出现大面积无职责空栏，或为了铺满裁掉任务要求的完整人物，均不通过。允许不同视口采用不同分区、裁切版本和界面关系。

用户指出文字过小时，在实际页面枚举当前视口中可见的文本节点并读取 computed `font-size`、`line-height`、裁切和对比度。用户明确说明示例不完整时，审计同类导航、状态、标签、按钮和正文，而不是只改被圈出的几个选择器；用户明确要求其它文案不动时，只调整排版属性，不顺手改写文字。源码搜索固定字号只能帮助定位，不能代替最终可读性检查。

对含推入、弹出、放大、跟随或悬浮定位的元素，分别截图或读取几何：

| 阶段 | 检查内容 |
| --- | --- |
| 动画前 | 元素是否位于合同起点，是否仍错误命中 |
| 动画中 | 运动方向、遮罩和层级是否正确 |
| 动画后 | 相对视口、容器、基线或相邻元素的终点偏差 |

布局外层、动画内层和命中层应各自保持职责。终点检查读取实际 `getBoundingClientRect()` 或等价几何，不用“已经能看见”“大致居中”代替。桌面、窄屏和移动端分别核对，因为同一动画百分比可能在不同容器尺寸下产生不同终点。

## 6. 验证互动映射

在互动区域的边缘、中心和多个代表点使用真实指针操作，同时记录：

- 输入坐标；
- 实际取样坐标；
- 反馈区域几何；
- 反馈内容对应的源位置；
- 离开生效区后的状态；
- `elementFromPoint()` 和实际命中元素。

如果反馈应出现在独立观察区，确认它没有跟随指针覆盖原人物；如果反馈应在原位置，确认它不会离开边界或遮挡必要信息。

再用触摸或键盘完成同一信息获取。悬停不能成为触摸设备唯一入口。链接、按钮、遮罩和底层内容分别做坐标点击，视觉范围与命中范围必须一致。

## 7. 验证循环和转场

在最终页面尺寸连续观看多轮，并在边界前后逐帧或慢速查看：

- 主体轮廓是否出现双重影像；
- 背景线条、光线和相机是否跳变；
- 动作反向是否符合物理和表情语义；
- 切点是否出现停顿、黑帧或亮度叠加；
- 播放器是否在边界重新请求或短暂显示 poster；
- 页面其它界面是否被媒体滤镜或混合影响。

编辑器里使用了淡化、光流或补帧不代表通过。最终浏览器中能看见重影或跳变时，回到真实边界重新选循环方法。

## 8. 验证媒体选择和异步状态

通过所有正式入口逐项选择媒体，确认：

- 所有入口复用同一选择边界；
- 当前 ID、可见消费者和 `currentSrc` 一致；
- 备用消费者只在准备完成后晋升；
- 过期异步回调不会暂停、归零、隐藏或重播当前消费者；
- 任一稳定帧显示的媒体层数符合合同；
- 播放速度、静音、起播位置和完成策略来自正式清单。

如果存在目录、会话抽样或媒体结束后推进，分别核对完整目录、当前选择和临时样本，不用一个数组同时承担不同职责。

页面改名、合并或移除时额外核对：

- 页面注册真源只保留最终稳定身份和顺序；
- 旧导航、旧 URL、历史恢复和站内按钮不能再进入退役页面；
- 旧渲染分支、样式入口、媒体清单和预热任务不再可达；
- Network 中没有由正式页面请求退役媒体；
- 当前页面使用的截图、poster 和视频帧内部没有旧界面；
- 原页面承担的有效行动已迁移到合同目标，或按用户决定明确取消。

源码搜索只能证明部分调用点消失。必须从正式入口操作页面，并观察最终导航、请求和可见内容。

## 9. 验证滚动与章节

只有项目包含滚动演出时执行：

1. 使用真实滚轮、触摸和键盘走完整路径；
2. 记录进入、完整就位、演出中段、终态和释放；
3. 核对连续进度只有一个生产者；
4. 确认普通页面没有未经合同允许的长期输入拦截；
5. 反向返回符合当前合同，没有无视觉反馈的多余距离；
6. 减少动态效果模式移除只为动画存在的跑道。

验收脚本可以读取生产状态，不能直接设置进度、完成标记、媒体 `ended` 或目录选择来制造结果。

再分别执行导航生命周期：

1. 从非起始位置完整刷新，观察首次绘制和最终滚动位置；
2. 使用正式站内导航进入目标页面；
3. 使用浏览器前进和后退返回；
4. 浏览器支持页面缓存时检查 `pageshow` 恢复路径；
5. 核对每条路径的页面、滚动、URL、媒体和互动状态是否来自对应的唯一所有者。

完整刷新从开头开始，不代表同一会话的历史返回也必须清零；反过来也一样。若页面先归零再跳回旧位置，沿首次脚本、应用初始化、路由恢复和持久化消费者查找第二个位置所有者。

## 10. 验证传输、声音和可见性

- 媒体请求返回正确 MIME；需要 Range 时响应可用；
- 同一资源没有因重复预加载边界产生无意义请求；
- 无需声音的角色媒体没有意外音轨或全局开关污染；
- 页面进入后台或媒体完全离开使用范围后，按合同暂停；
- 返回时恢复当前正式状态；
- 减少动态效果模式使用 poster 或静态内容，并保留导航。

## 11. 验证透明媒体

透明媒体需要同时证明：

1. 高质量 Alpha 母版真实存在；
2. 网页输出能解码出有效 Alpha；
3. poster 与视频来自同一清理后结果；
4. 正式页面引用的是新输出；
5. 在浅色、深色和实际背景上没有黑底、绿边、错误预乘或全画布矩形。

只有实际观察到全画布低 Alpha 污染时才运行清理脚本。统计变化不能代替人物细边和真实背景上的视觉检查。

## 12. 真实故障结果

对可能失败的核心媒体使用真实不可用路径或浏览器条件检查：

- poster 和主要导航仍可见；
- 页面不会永久空白或锁死；
- 用户得到合同规定的重试、静态内容或可继续浏览结果；
- 恢复后仍使用同一正式选择和播放边界。

故障检查用于证明恢复行为，不把具体故障请求、参数或失败截图保存进 Skill。

## 13. 交付报告

按以下顺序：

```text
结论：
- 用户在目标视口能观察到什么

证据：
- 素材生产
- 选定视觉与全局品牌消费者
- 请求与解码
- 最终消费者
- 响应式构图
- 导航生命周期与动效终点
- 互动映射
- 循环或转场

未通过：
- 实际结果
- 影响
- 根因所在边界

未验证：
- 内容与原因

证据版本：
- 最后一次影响结果的修改
- 此后重跑的真实链路
```

不使用“运行正常”“已优化”或“测试通过”代替证据。验收结束后只关闭本次启动的服务和调试进程，不影响用户原本运行的进程。
