# 真实浏览器验收

用于证明角色网站真的启动、媒体真的传输、导演真的消费、用户真的看到结果。函数单测只能补充，不能代替本清单。

## 1. 验收前准备

确认：

- 使用的是最终网页媒体，不是原始生成文件或占位视频。
- 本地服务和生产环境采用与实际部署一致的媒体路径。
- 浏览器控制台、网络面板和媒体元素状态可观察。
- 测试会执行真实点击、滚轮、触摸或键盘操作。
- 当前工作区的用户改动已识别，不因验收覆盖或删除。

记录环境：

```text
地址：
浏览器与版本：
视口：
网络条件：
减少动态效果：开 / 关
声音：开 / 关
测试时间：
```

如果是已有网站迁移或局部改版，同时记录：

```text
要求保持不变的页面和状态：
加载态基准截图：
桌面首屏基准截图与视口：
移动首屏基准截图与视口：
关键互动基准录屏或说明：
允许重新设计的区域：
```

先检查保护区，再检查新功能。React、Vue、GSAP 或其它实现改变不能作为视觉变化的理由。

## 2. 素材生产边界

对媒体清单中的每个文件检查：

- 文件存在且非零。
- 浏览器支持编码格式。
- 分辨率、帧率和像素格式一致或有明确原因。
- 实际时长与媒体清单一致。
- 裁剪后的动作高潮和片尾都可用。
- 锚点图尺寸和页面比例正确。

可用 `ffprobe` 获取确定性信息，但最终仍需播放检查画面。

## 3. 传输边界

启动网站后检查网络请求：

- 锚点图返回成功并在首个视频前可见。
- 首个待机视频返回 200 或按 Range 返回 206。
- 服务器提供正确的 `Content-Type: video/mp4`。
- Range 响应包含合理的 `Accept-Ranges` 和 `Content-Range`。
- 后续视频只按导演计划预加载，没有加载同一素材的无限请求。
- 文件 404 或解码失败时，启动屏不会永久遮挡，锚点和导航仍可用。

## 4. 首次加载

步骤：

1. 以新页面打开网站，不先点击。
2. 在启动界面消失前截取加载态，检查标志、文案、进度、背景和退场时机。
3. 等启动界面结束。
4. 读取当前视频的 `src`、`readyState`、`paused`、`currentTime` 和 `duration`。
5. 观察几秒，确认 `currentTime` 持续增长。

通过标准：

- 锚点先提供完整首屏，不出现空白背景。
- 首个待机实际 `play()`，不是只设置了文件名。
- 导航没有遮挡角色头部或关键特征。
- 文案和按钮可读，不随视频模糊。
- 被标记为保留的加载动画和首屏，在相同视口、相同媒体状态下与基准的构图、裁切、遮罩、标题、控件和文案一致。

加载态、转场峰值、视频结束态等瞬时界面必须在真实时机观察。只在页面稳定后读取 DOM，不能证明这些状态没有被改坏。

## 5. 自主表演

步骤：

1. 不操作页面，观察至少一轮计划序列。
2. 记录每次片段键名、开始时间和去向。
3. 在切换点观察人物、锚点和媒体舞台。

通过标准：

- 顺序符合体验合同。
- 随机动作不会连续重复。
- 备用播放器在切换前已经可用。
- 锚点没有清晰停留成“图片—视频—图片”的卡顿循环。
- 没有黑帧、双人重影或转场外露出的硬切。
- 切换结束后媒体舞台恢复清晰，导演的当前键名与可见视频一致。

## 6. 偶然点击与明确操作

分别执行：

1. 点击角色画面。
2. 点击真正空白区域。
3. 点击导航、标题文案、链接、动画按钮和随机按钮。
4. 在弹窗打开时点击弹窗内外。

通过标准：

- 角色和空白区域触发约定的直接回应。
- 导航、文案和按钮不会附带触发空白点击回应。
- 一个点击只产生一个最高优先级请求。
- 明确按钮播放指定动作或执行未来替换后的功能。
- 叙事锁定期间其它操作不会抢走视频。
- 返回欢迎或自动动作正在转场时点击明确按钮，用户最后一次有效操作会在当前转场后执行，不会被静默丢弃。

## 7. 第一次滚动剧情

这是关键测试，不使用消费端手写 `ended = true` 或直接调用滚页函数。

步骤：

1. 刷新后停在首屏顶部。
2. 发送一次真实向下滚轮。
3. 记录剧情片段的 `duration` 和开始时间。
4. 播放期间连续发送多次向下滚轮。
5. 在片尾读取 `currentTime`、`ended` 和页面 `scrollY`。
6. 等浏览器实际触发 `ended`，观察页面何时滚动。

通过标准：

- 第一次有效滚轮启动剧情片段。
- 切换完成前没有提前滚动。
- 播放期间连续滚轮均被锁住，首屏没有逐渐露出下一章节。
- 页面开始滚动时，目标播放器满足 `ended === true`，且 `currentTime` 接近真实 `duration`。
- 导演没有在片尾前切回待机。
- 动画失败或超时时停留在首屏、恢复输入并提供重试，不假装成功滚页。

触摸端重复测试：从首屏向上滑超过阈值，播放期间继续滑动，结果应与滚轮一致。

## 8. 离开和返回

步骤：

1. 正常进入下一章节。
2. 确认首屏视频暂停。
3. 通过导航返回首屏。
4. 再通过下方动作入口返回首屏并要求播放指定动作。

通过标准：

- 离开时不会继续无意义解码和自动切片。
- 普通返回只触发一次欢迎回应。
- 指定动作返回时不会同时触发欢迎回应。
- 返回后的可见视频、导演键名、按钮状态和进度一致。

## 9. 声音、页面可见性与错误恢复

测试：

- 默认静音能自动播放。
- 开启声音后两个播放器都同步状态。
- 页面切到后台，当前视频暂停；回到前台按当前状态恢复。
- 在切换时模拟一个真实视频 404 或不可解码文件。

通过标准：

- 自动播放策略失败不会造成永久停住。
- 新视频失败时恢复旧视频或锚点，媒体舞台不永久模糊。
- 导演解除切换锁，可以继续接受有效请求。
- 控制台报告可理解的错误，用户仍可浏览页面。

## 10. 响应式和减少动态效果

至少检查：

- 常见桌面视口。
- 约 390×844 的移动视口。
- 粗指针或真实触摸模式。
- `prefers-reduced-motion: reduce`。

通过标准：

- 角色脸、头部和关键动作没有被导航或裁切破坏。
- 文字和控制不会盖住角色主要表演区域。
- 移动端无需悬停即可访问主要功能。
- 减少动态效果模式显示静态锚点，导航和章节仍可使用。
- 已有移动端控制、文案和互动不能因为“简化移动布局”被擅自隐藏；只有变更合同允许时才调整。

## 11. 后续章节的信息、动态和滚动成本

逐章走真实滚动链路：

1. 写下该章的信息任务、主要内容证据，以及角色或风格 DNA 如何延续。
2. 在桌面和移动端记录 `section.offsetHeight / window.innerHeight`，与章内推进合同的预计屏数对照。
3. 使用真实滚轮或触摸进入章节，分别截取舞台刚固定、一个或多个有意义的中间状态、接近退出时的画面。
4. 在作品章节确认真实项目名称、说明、实现亮点或结果证据和有效入口都能直接看到并使用。
5. 检查独立图片、装饰组件、CSS 伪元素和动画调用点，逐项说明其角色身份、内容、结构或交互职责。

通过标准：

- 除非合同明确声明有意停留，否则不应连续滚动约一屏仍没有新的信息、构图关系或状态。
- 进入舞台和章内推进使用独立进度边界；压缩章节后，内部动画不会在舞台固定前结束。
- 任意抽样进度都有清楚焦点，没有空白失焦、重复状态或用漂浮素材填时间的段落。
- 角色或风格 DNA 延续到后续章节，但没有遮住真实内容证据。
- 首屏门控结束后使用原生滚动；没有全局滚轮加速、长期 `preventDefault()` 或强制翻页掩盖章节过长。
- 独立装饰与相邻内容存在可见关系，移除会造成明确的信息、结构或交互损失；否则不应出现在 DOM、素材清单和动画调用点。
- 减少动态效果模式不保留没有信息价值的固定距离，静态内容顺序仍完整可读。

只验证选择器存在、图片成功加载或动画函数被调用，不能证明用户得到这些结果。

## 12. 透明素材、内容和外部入口

透明图片和视频需要三层证据：

1. 文件层：真实存在 Alpha 通道，不是绿色背景或棋盘格图案。
2. 解码层：使用正确解码器读取 Alpha 平面，确认同时存在透明和不透明像素。
3. 页面层：在目标浏览器和真实章节背景上播放或显示，没有黑底、绿边、错误预乘和明显溢色。

文件扩展名、编码标签、查看器背景颜色和单张封面都不能单独作为透明通过证据。不支持当前透明编码的浏览器必须显示约定的海报或其它回退。

内容区同时检查：

- 项目、博客和社交入口对应真实地址。
- 尚未提供的地址明确显示“待接入”，不生成看似真实的假链接。
- 场景道具没有被误写成角色原设。

## 13. 视觉保护区对照

对每个标记为“保留”的状态，在相同视口和尽量相同媒体帧下对照：

| 状态 | 对照内容 |
| --- | --- |
| 加载态 | 标志、文案、进度线、背景、最短展示时间、退场 |
| 桌面首屏 | 角色裁切、导航、标题、遮罩、按钮、控制组、滚动提示 |
| 移动首屏 | 角色焦点、标题顺序、动作按钮、声音和目录入口 |
| 关键互动 | 触发方式、动画、结束去向和返回状态 |

功能测试全部通过但保护区截图明显改变时，验收仍然失败。先恢复保护区，再继续新页面工作；不能把新的审美判断当成默认授权。

## 14. 运行环境清理

记录本次验收启动的开发服务器、浏览器调试实例和端口。交付前：

1. 停止本次启动且不再需要的服务和调试进程。
2. 检查默认端口是否仍在监听。
3. 如果用户自己的服务原本就在运行，不擅自结束；说明复用了哪个进程。
4. 从干净状态实际运行一次用户启动脚本，确认不会因为验收进程占用端口而失败。
5. 不向用户声称“可以直接启动”，除非默认端口已经释放或明确说明已有同一网站正在运行。

## 15. 验收报告

按以下格式交付：

```text
通过：
- 用户可观察结果 + 实际证据
- 后续各章节的实际屏数和开始 / 中段 / 结束画面证据

失败：
- 复现步骤
- 实际结果
- 期望结果
- 根因所在边界：素材 / 传输 / 导演 / 事件 / 布局

未验证：
- 未验证内容
- 原因
- 需要的条件

清理：
- 已停止的本次进程
- 保留的用户进程及原因
- 默认端口最终状态
```

不要只写“运行正常”“测试通过”。关键结论要带真实片段、媒体事件、滚动位置或浏览器状态证据。
