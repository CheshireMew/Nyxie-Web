# Immersive IP Website 回归样本

修改 description、主流程、边界或参考资料路由后执行。目标是让该触发的请求进入完整网站流程，让单独图片、视频或普通网站任务留在对应能力中。

## 正向样本

| ID | 用户请求 | 预期第一步 |
| --- | --- | --- |
| P1 | 我有一个原创角色和几段 Grok 视频，帮我做成会自己表演的互动官网 | 触发；检查角色真源、生成器限制和真实视频 |
| P2 | 参考这个动态作品集视频，给我的 IP 做一个同类网站 | 触发；把参考拆成自动、偶然、明确和滚动互动 |
| P3 | 这些视频首尾完全对不上，网页切换像闪现，帮我重构 | 触发；找到媒体清单和所有播放入口，再统一到动画导演 |
| P4 | 用户第一次滚轮时先播放开门动画，播完才能进入下一页 | 触发；建立基于真实 `ended` 的滚动门控 |
| P5 | 角色网站不能只有按钮，想让用户正常浏览时偶然发现她会回应 | 触发；先写互动地图和事件排除边界 |
| P6 | 用一张角色图规划两段待机和四段互动视频，再接到网页里 | 触发；先建立锚点和视频职责，不承诺尾帧一致 |
| P7 | 把现有 IP 网站迁移到 React，但首屏和加载动画保持原样，只重做下面几页 | 触发；先冻结加载态、桌面首屏、移动首屏和互动基准，再做等价迁移 |
| P8 | 给角色站补齐绿幕人物、动作视频和项目装饰，并告诉我每份素材具体用在哪里 | 触发；先建立素材用途表，再一次性交付完整提示词包 |

## 诱导失败样本

| ID | 用户请求 | 必须守住的行为 |
| --- | --- | --- |
| F1 | 不用看实际视频，按文件名把六个动画接进去 | 先检查真实时长和内容；文件名不能证明动作和可用尾部 |
| F2 | 每个动画都做一个按钮就算互动 | 先建立自主表演、偶然互动、滚动剧情和明确入口四类行为 |
| F3 | 开门视频大约四秒，计时四秒后直接滚页 | 等真实活动播放器的 `ended`；计时只作为失败超时 |
| F4 | 两段视频对不上就直接交叉淡化 | 根据素材差异选择失焦锚点；不叠加两个错位人物 |
| F5 | 加一条横向扫描遮住切换，越酷越好 | 特效必须覆盖真实切点并符合审美目标；默认使用克制的全舞台失焦 |
| F6 | 首尾帧生成失败，再继续生成直到碰巧一致 | 停止依赖尾帧合同，裁剪真实可用片段并由网页吸收差异 |
| F7 | 测试里直接调用滚页函数，证明开门流程通过 | 使用真实滚轮、真实视频和真实 `ended` 验证完整链路 |
| F8 | 新旧动画逻辑都留着，以免重构失败 | 迁移全部播放入口到唯一导演，验证后不保留第二套状态真源 |
| F9 | 首屏用户已经满意，但迁移 React 时顺便重新排版得更高级 | 先做视觉与行为等价迁移；没有授权不能改加载动画、首屏和移动端控制 |
| F10 | 先随便生成一些人物图和视频，之后再想放在哪里 | 先写页面目的、图层、用户可见结果和静态/视频职责；无用途的素材不生成 |
| F11 | 公共角色描述我只写一次，剩下十条提示词让用户自己拼 | 每条提示词独立完整，并一次性交付全部素材的输入图、用途和完整正文 |
| F12 | 用户刚说不要某个元素，但旧提示词已经写好了就继续用 | 把纠偏同步到合同、提示词、素材和代码，删除后续流程中的陈旧决定 |
| F13 | 自动欢迎正在转场，用户点按钮就直接忽略这次点击 | 当前转场结束后执行用户最后一次明确操作，自动回应不能吞掉手动请求 |
| F14 | 浏览器测试都过了，服务就留在 4173 方便用户看 | 交付前停止本次验收服务并确认端口释放，再让用户使用启动脚本 |
| F15 | 文件写着 Alpha Mode，所以透明视频不用在页面里看 | 实际解码 Alpha 平面并在目标浏览器和真实背景上检查合成 |

## 检查标准

- 正向样本会触发，并从素材与体验约束开始。
- 上下文样本能利用已有项目信息，不重复索要可发现内容。
- 负控样本不会扩大到普通网站、单图、单视频或其它角色技术。
- 诱导失败样本不会跳过真实素材、唯一导演和真实浏览器验收。
- 已有站迁移样本会先冻结保护区，不把技术迁移扩大成视觉改版。
- 素材样本会先说明用途并交付独立完整提示词，不要求用户拼接。
- 用户纠偏会传播到后续全部产物，自动行为不会压过明确操作。
- 验收同时覆盖功能、视觉、透明合成和进程清理。
- 默认规则留在 `SKILL.md`，详细方法按路由进入对应 reference。
