# 沉浸式 IP 网站项目简报

用于从零规划、重大改版或素材方向尚未确定的任务。能从项目、参考和真实媒体中确认的内容由 AI 先填写，只把会改变结果且无法从现有材料判断的决定交给用户。

## 3. 信息架构与页面目标

先判断页面是否需要存在：

| 页面或候选页面 | 独立信息、探索结果或行动 | 内容证据 | 与其它页面的重叠 | 决定与依据 |
| --- | --- | --- | --- | --- |
| | | | | 保留 / 合并 / 改名 / 重做 / 移除 / 待确认 |

再记录稳定页面身份。显示名称可以改，页面 ID、顺序、渲染、导航、历史和预热必须从同一份注册真源派生：

## 5. 候选设计方向

只有方向尚未确定时填写二到三个：

| 方向 | 人物与环境关系 | 页面构图 | 互动方式 | 媒体生产方式 | 响应式代价 | 适合当前目标的原因 |
| --- | --- | --- | --- | --- | --- | --- |
| | | | | | | |

候选需要在结构上不同，不能只是更换颜色名称或形容词。用户确认后只保留选中方向进入后续合同；未选方向不进入提示词和实现。

选定方向进入实现前记录忠实边界：

| 选定稿 | 必须保留的轮廓、比例、负空间、层级和色彩 | 允许局部调整 | 不得自行重画 | 实际显示尺寸 |
| --- | --- | --- | --- | --- |
| | | | | |

“采用这个方案”表示按选定稿落实上述关系，不表示只借用题材或风格重新设计。用户只点名一个局部变化时，更新该变量和受它直接影响的几何，不扩展为整套重画。

全局品牌识别另外盘点消费者：

| 规范母版 | 页眉 / 导航 | favicon / 应用图标 | 加载界面 | 元数据 / 分享图 | 烘焙媒体 | 派生与缓存策略 |

### 6.1 媒体集合构图合同

页面同时展示多项媒体时填写。集合先有排布规则，单项卡片再从规则获得尺寸和位置。

| 决定 | 当前结论 | 唯一真源 |
| --- | --- | --- |
| 正式条目、顺序和当前数量 | | 内容目录 |
| 预计支持的数量范围 | | 用户目标与维护方式 |
| 每项原始长宽比与完整范围 | | 真实媒体 |
| 显示比较基准 | 等高度 / 等宽度 / 等视觉面积 / 有意主次 | 页面目标 |
| 是否允许裁切 | | 构图合同 |
| 分行、分列或轨道函数 | | 容器、比例与数量 |
| 水平和垂直间距 | | 目标视口 |
| 允许遮挡及不得遮挡的内容 | | 信息与互动职责 |
| 当前选择、读数和行动入口 | | 互动合同 |
| 增加一项和两项后的结果 | | 同一排布函数 |

逐项绝对定位只有在固定拼贴本身是设计目标、数量边界已确定且超出边界有回退时才成立。否则记录集合级分组规则，不为当前每项分别指定坐标。

## 13. 验收合同

```text
[ ] 页面目标由用户可观察结果定义，没有使用角色标签代替
[ ] 每个保留页面都有独立信息、探索结果、内容证据或行动
[ ] 页面身份、顺序、渲染、导航、历史和预热来自同一真源
[ ] 人物、动作、环境、网站视觉和文案各有真源
[ ] 设计方向在制作前已沟通并确认，或用户已授权 AI 直接选择
[ ] 选定视觉稿的轮廓、比例、负空间、层级和色彩在实现前已锁定
[ ] 用户只要求局部修改时，其它已确认视觉关系保持不变
[ ] 全局品牌资产的页眉、favicon、应用图标、加载界面和元数据消费者已同步或明确排除
[ ] 素材完整展示范围、人物位置、场景关系和界面安全区已确定
[ ] 超宽屏、常规桌面、窄屏和移动端都有意图明确的构图
[ ] 混合比例媒体已经分开记录原始比例、显示尺度、裁切和视觉权重
[ ] 媒体集合由数量、比例、容器和视口共同排布，没有只适配当前条目的逐项坐标
[ ] 用户要求继续增加内容时，增加一项和两项仍沿同一规则获得可读、可达的结果
[ ] 当前选择、状态、行动入口和其它有效信息优先于无职责装饰文案
[ ] 用户实际低高度窗口使用真实可视高度，全屏章节没有被固定最小高度撑大
[ ] 一屏展示的内容通过重排和缩放完整呈现，没有用新增滚动条代替适配
[ ] 原媒体负空间和页面新增空间都有职责
[ ] 互动的输入、取样和反馈位置已分别定义
[ ] 离散滚轮按手势而非单个事件推进，惯性尾段和首尾释放已有合同
[ ] 位移动效的最终布局、动画运动和命中区域由清楚的所有者负责
[ ] 用户要求调整速度的阶段有唯一时间源，真实入口的触发、首个反馈和终态时间符合合同
[ ] 真实视频已检查可用区间、循环边界、音轨和编码
[ ] 循环或转场在最终尺寸中没有重影、双重轮廓和明显跳变
[ ] poster 在最终视频首帧可见前持续提供完整画面
[ ] 预加载由正式选择逻辑触发，并复用最终消费者或明确缓存边界
[ ] 导航目标、可选过场和下一页预热分别有唯一所有者，跳过过场不影响预热
[ ] 即将进入视口的最终节点已挂上目标源并解码，而不是只在独立预加载器中完成下载
[ ] 真实请求、解码、显示和互动结果经过生产代码得到
[ ] 完整刷新、站内导航、前进后退和页面缓存恢复分别符合合同
[ ] 改名、合并或移除的页面不再能通过旧入口渲染、预热或请求
[ ] 正式截图、海报和视频画面没有被遗漏的过期界面或文案
[ ] 减少动态效果模式保留完整信息和导航
[ ] 保留区域与基线一致
[ ] 旧监听、旧状态、旧 helper 和旧恢复路径已从有效实现移除
[ ] 所有相关验收证据均在最后一次影响结果的修改之后生成
```

## 14. 分工

用户决定角色事实、审美取舍、不能接受的表现、付费生成操作和发布范围。AI 负责从已给材料提取事实、显式标记创作决定、提出可比较方向、设计精确提示词、处理授权素材、实现网站并提供真实验收证据。
