每章都要回答“为什么必须有这一章”。角色站不能只在首屏出现角色，后续全靠文字和装饰道具支撑；关键章节至少要有角色本体、角色局部、角色动作或角色明确留下的视觉信息。

### 章内推进合同

| 章节 | 信息任务 | 主要内容证据 | 角色或风格延续 | 开始 / 中段 / 结束状态 | 预计屏数 | 可移除项 |
| --- | --- | --- | --- | --- | --- | --- |
| | | | | | | |

- 作品章节的主要证据至少包括真实项目名称、说明、实现或结果证据，以及有效入口；角色和装饰不能替代这些内容。
- 每个可见状态都要增加信息、构图关系或明确反馈。没有新状态的滚动距离应缩短。
- 留白可以用于聚焦和节奏，不为填满画面增加无关系的首饰、星月、粒子或悬浮物。
- 角色延续不等于每章摆一张立绘；它可以通过角色动作、视线、局部、叙事痕迹和同一套视觉语言建立。

## 11. 验收合同

```text
[ ] 首个待机实际启动
[ ] 自动表演至少完成一轮
[ ] 偶然互动与明确按钮不会互相误触发
[ ] 视频切换没有清晰静态停顿、黑帧或双人重影
[ ] 章节视频真实 ended 后才滚动
[ ] 连续滚轮不会提前离开首屏
[ ] 后续章节恢复原生滚动，没有全局滚轮倍率或长期输入拦截
[ ] 后续章节的进入时间线与章内推进时间线相互独立
[ ] 各章节开始、中段和结束状态都有可见信息或构图推进
[ ] 桌面与移动端的实际滚动屏数符合合同，没有长距离空滚
[ ] 作品区可见真实项目、说明、实现或结果证据和有效入口
[ ] 独立装饰通过角色身份、内容、结构或交互职责检查，移除后果明确
[ ] 返回首屏状态正确
[ ] 桌面、移动和减少动态效果模式可用
[ ] 真实视频通过浏览器加载，核心链路没有 mock
[ ] 所有保留区域的加载态、桌面态、移动态和关键互动与基准一致
[ ] 场景道具没有未经确认升级成角色核心设定
[ ] 透明素材在真实页面背景上正确合成
[ ] 外部链接真实存在，缺失项明确标记待接入
[ ] 验收后已停止本次服务和调试进程，默认端口可重新启动
```

## 12. 用户与 AI 分工

用户负责决定：

- 角色身份和不可改变的设定。
- 审美方向、不能接受的表现和最终素材取舍。
- 生成服务的登录、次数和付费操作。
- 是否发布以及发布到哪里。

AI 负责：

- 从参考和项目中提取可执行结构。
- 设计精准提示词和素材清单。
- 检查真实媒体并提出裁剪、重生或改职责建议。
- 建立网站唯一媒体清单与动画导演。
- 完成真实浏览器验收并报告证据。
