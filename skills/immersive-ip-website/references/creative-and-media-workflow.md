# 角色视觉与视频素材工作流

用于确定角色网页的艺术方向，编写图片或视频提示词，检查真实成片，并准备能在页面中稳定使用的媒体。

## 1. 先回答页面为何需要这份媒体

在讨论艺术风格或动作前写清：

```text
页面目的：
媒体在页面中的职责：
访问者第一眼看到什么：
访问者互动后多看到什么：
必须完整展示的范围：
媒体与相邻界面的关系：
如果移除这份媒体，页面会失去什么：
```

说不清职责的素材不进入生成计划。留白可以承担聚焦、节奏和界面空间，不需要靠无关道具或粒子填满；没有职责的空区同样不能只因为它存在于原素材中就原样保留。

同时确定现有素材的使用边界：

| 素材 | 当前页面职责 | 本轮决定 | 可以出现在哪里 | 必须退出哪些生产引用 | 实体文件权限 |
| --- | --- | --- | --- | --- | --- |
| | | 继续使用 / 仅作参考 / 替换 / 归档 / 删除 | | | 已授权 / 未授权 |

“不再使用旧素材”表示正式页面、媒体清单、预热、poster、回退、测试输入和文档示意不再把它当成有效产物。原文件是否归档或删除是另一项权限决定；未获授权时可以保留在非活动位置，但不能保留活动引用或兼容入口。

## 2. 分开记录事实、参考与创作授权

| 视觉决定 | 真源 | 只借用什么 | 未决定什么 |
| --- | --- | --- | --- |
| 人物身份 | 用户指定的设定图 | 可见脸部、发型、身体特征、服装、材质、原设配饰 | 动作、性格、环境、故事 |
| 动作与神态 | 用户确认的动作参考，或明确的创作授权 | 重心、动作线、视线、手势、动作瞬间 | 人物身份和场景 |
| 环境与光线 | 用户确认的环境参考，或明确的创作授权 | 空间、材质、前中后景、主色关系、光线 | 人物身份和文案 |
| 网页构图 | 当前网站、真实容器和互动合同 | 人物位置、占比、完整范围、文字与互动安全区 | 新角色设定 |

旧文案、形态名称、服装颜色和模型偶然生成物不能补齐空缺。网站配色可以约束色彩关系，但不能凭空决定空间类型；角色设定可以锁定身份，但不能自动决定动作。

如果用户要求 AI 提候选方向，先提出真正不同的画面关系并标记为待确认。候选应说明人物如何占据空间、环境如何回应动作、界面在哪里参与，不使用一组风格形容词冒充差异。用户确认或明确要求 AI 直接选择后，才把方向写进提示词。

### 2.1 忠实落实选定视觉

用户选定某一候选稿、参考图或生成结果后，把它从“候选”提升为当前视觉真源。先记录被接受的关系：

```text
外轮廓与长宽比例：
内部关键折点、缺口和负空间：
主次笔画或形体层级：
颜色分别落在哪些局部：
文字与图形的相对位置：
实际页面中的显示尺寸：
用户允许改变的局部：
```

从位图转成 SVG、CSS 几何或组件时，先沿上述关系描摹，不用“同一概念”替代忠实还原。比较时同时看放大轮廓和页面实际尺寸：放大图暴露折点、圆弧、切口和笔画关系，实际尺寸暴露粘连、过细、失真和识别度。局部调整只改变用户点名的端点、长度、位置、颜色或间距，并确认未连带改变其它锁定关系。

## 3. 从原图提取可见身份

只记录能从图中确认且会影响一致性的内容：

```text
脸部与眼睛：
发型与头部轮廓：
身体比例允许怎样变化：
服装轮廓、层次和材质：
原设配饰：
图中实际存在的其它特征：
图中没有决定的内容：
```

不要从这些内容推断职业、性格、固定姿势或生活环境。生成结果新增的物件、建筑、植物、图案或光效仍是当前画面变量；用户没有确认前，不进入角色设定、网页文案或后续提示词。

多张参考图分别标记职责。不同图片之间不自动继承主题，也不根据文件排序、文件名或历史生成结果补写设定。

## 4. 把审美判断变成可画的关系

艺术方向至少明确以下关系：

- 人物在画面中的位置、占比和完整范围；
- 支撑点、重心、身体方向、视线和手势；
- 人物与场景中的表面、光线、前景或空间边界如何发生关系；
- 环境主色、人物主体色和局部强调色分别占据哪里；
- 主光方向、明暗层级和焦点顺序；
- 桌面、超宽屏、窄屏和移动端需要保住的区域；
- 文字、导航、细节观察或其它互动反馈的安全区。

“有张力”要落到可见关系：不对称的重心、清楚的动作方向、尚未完成的动作瞬间、视线目标和有用途的负空间。它不等于固定使用扭身、回看镜头、前伸手或强透视。

“与网站风格一致”要落到色彩比例、材质、字体和图形关系。角色身上的局部强调色不会自动扩散为整幅环境色；环境颜色应来自网站真源或当前环境决定。

### 4.1 分开原始比例、显示尺度和视觉权重

媒体进入页面前分别记录：

| 变量 | 由什么决定 | 它不代表什么 |
| --- | --- | --- |
| 源文件分辨率 | 文件真实宽高像素 | 页面应该显示多大 |
| 原始长宽比 | `naturalWidth / naturalHeight` 或媒体元数据 | 卡片必须统一宽度或高度 |
| 裁切与适配 | 必须完整显示的内容和容器职责 | 可以随意改变原始比例 |
| 页面显示尺度 | 最终渲染宽度、高度和面积 | 源文件物理尺寸 |
| 视觉权重 | 显示面积、明暗、对比、位置和内容密度 | 单一 CSS 宽度 |

用户要求“按图片实际比例”时，默认保持原始长宽比并按页面合同决定显示尺度，不把源像素一比一映射为页面尺寸。用户说某张看起来太大或太小时，先读取最终 `getBoundingClientRect()`、可见媒体区域和相邻项，判断问题来自显示高度、宽度、面积、裁切还是位置，随后只改变对应变量。

同一集合包含横图、竖图和方图时，先选择比较基准。等显示高度通常能保持原始比例并让一组图片拥有可比较的垂直尺度；等显示宽度、等视觉面积或有意主次也可以成立，但必须来自页面目标。卡片边框、标题和说明在媒体尺寸之外计算，不能通过拉伸图片让外框一致。

单张媒体的尺寸决定不能脱离整个集合。用正式条目数量、媒体比例、可用容器和目标视口计算分组、单项尺度与间距，并检查新增内容后的结果。某一项若因横竖比例在统一规则下显得意外弱小，应调整集合比较基准或分组关系，不为它增加一个只在当前屏幕成立的特例尺寸。

## 5. 决定整景媒体还是分层媒体

| 条件 | 更适合整景图片或视频 | 更适合人物与背景分层 |
| --- | --- | --- |
| 人物与环境关系 | 动作、光线和空间必须作为一幅完整镜头成立 | 人物需要在不同界面和比例中重新排布 |
| 响应式范围 | 目标视口接近，允许受控裁切 | 超宽屏、桌面和移动差异很大 |
| 互动 | 互动读取完整镜头中的位置 | 背景、人物和反馈需要独立响应 |
| 制作成本 | 已有高质量完整成片 | 有可靠透明素材或可单独生产 |

整景媒体更省合成工作，但把人物、环境和负空间绑定在一起；分层媒体更灵活，但需要处理边缘、光线和接触关系。选择基于当前页面目标，不把任一方式写成全站默认。

对整景媒体，网页排版应利用原始构图：信息、观察窗或导航进入真实负空间；如果原媒体与目标视口不匹配，可以使用受控裁切、局部舞台、跨栏排版或另一份裁切版本。不能为了铺满屏幕裁掉必须完整展示的人物，也不能在媒体旁留下没有职责的空栏。

## 6. 生成首帧

首帧需要同时服务静态展示和后续运动：

1. 人物身份来自设定真源。
2. 动作来自已确认参考或本轮授权的创作方向。
3. 环境和光线来自环境真源。
4. 比例、人物位置和安全区来自真实网页舞台。
5. 画面停在动作已经有方向、但尚未完成的瞬间。
6. 主要轮廓和任务要求的完整范围在画内，并为动作方向留下运动空间。
7. 次级元素只服务主要动作、空间层级或网站构图。

安全区不是固定百分比。先根据人物完整轮廓、动作包络、相机运动和页面裁切计算，再在目标视口中预演。

## 7. 编写精确提示词

一份图片提示词按以下顺序组织：

```text
[角色身份与必须保持的可见特征]
[人物比例、位置、完整范围]
[动作的支撑、重心、动作线、手势和视线]
[人物与环境的具体空间关系]
[环境的前中后景、材质和主光]
[渲染方式、镜头和景深]
[网站色彩关系与界面安全区]
[少量会导致身份错误或网页不可用的边界]
```

一份视频提示词按以下顺序组织：

```text
保持首帧中的人物身份、服装、材质、环境和构图连续。
主要动作：
动作的起点、发展和完成状态：
允许的次级运动：
相机运动：
结束时保持的自然状态：
少量必要边界：
```

每条提示词只承担一个画面目标、一个主要动作、一个环境和一个主光关系。独立完整不等于重复堆叠；负面内容只保留身份漂移、参考职责混用、文字水印、身体结构或裁切会让素材无法使用的少量边界。

结果偏离时回到来源表查缺失决定，不在旧提示词末尾累积否定句，也不把偏离结果保存成后续默认。

## 8. 检查真实视频

先用 `ffprobe` 读取确定信息，再逐段播放：

```powershell
ffprobe -v error -show_entries format=duration:stream=codec_name,width,height,r_frame_rate,pix_fmt -of json input.mp4
```

至少观察开始、四分之一、中点、四分之三和结束附近，并记录：

```text
文件：
分辨率、帧率、时长和音轨：
人物实际位置与完整范围：
主要动作发生区间：
可用起点与终点：
镜头或身份变化：
首尾边界差异：
桌面与移动裁切风险：
适合承担的页面职责：
```

提示词不能证明成片中真的发生了什么。最终网页职责、裁切区间、速度和循环方式只由真实文件决定。

## 9. 选择循环和转场

逐帧比较候选起止点，先判断动作语义，再选择方法：

- 首尾姿态、光线和相机接近时，可以测试直接循环。
- 动作可自然往返时，可以测试往返播放；反向会让物理、表情或环境变化违和时不用。
- 存在相近动作相位时，可以寻找运动匹配切点。
- 必须隐藏不连续边界时，使用与页面视觉语言一致的短暂遮蔽或换场。

交叉淡化只有在两个时刻叠加后仍保持单一清楚轮廓时才可用。出现双重人物、边缘重影、背景错位或亮度叠加，就放弃该方法，不通过延长淡化掩盖。

循环验收要在最终浏览器、实际尺寸和真实背景上连续观看多轮；编辑器时间线顺滑不等于网页循环顺滑。

## 10. 准备网页媒体

根据目标浏览器、透明需求和文件体积决定格式。常见交付可以包含：

- MP4：兼容回退；
- WebM：在目标浏览器确有体积、编码或透明优势时使用；
- poster：与视频同构图的静态首帧；
- 移动端裁切版本：只有单一媒体无法保住关键构图时增加。

无需声音的角色视频移除音轨。编码后重新检查时长、尺寸、帧率、颜色、画面和浏览器播放，不能把命令成功当作通过。

截图、poster 和视频帧可能已经烘焙导航、标题、按钮、旧角色状态或其它界面。页面改名、合并、移除或更换设计语言时，要逐项检查这些可见内容；源码、DOM 和媒体文件名都已更新，不能证明画面内部没有旧界面。发现过期内容时重新生产、受控裁切或停止使用该媒体，具体方式由页面构图合同决定。

标志和站点图标优先保留一个矢量规范母版。页眉、favicon、应用清单、加载界面、分享图和烘焙媒体先登记为消费者；同一矢量能满足时直接共用，尺寸或背景要求不同才从母版派生。favicon 等小尺寸输出需要在常见标签尺寸下检查笔画和负空间，并通过文件指纹、文件名或版本参数使浏览器实际读取新版本，不能只确认源文件已经改写。

透明媒体保留高质量 Alpha 母版，网页输出和 poster 从同一母版生产。只有出现全画布低 Alpha 污染时才使用 `../scripts/clean_alpha_video.py`；阈值按真实素材检查，并在目标背景上确认发丝、薄纱和其它真实半透明细节没有受损。
