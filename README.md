# Nyxie Wonderland

夜希（Nyxie）的交互式角色展示网站。首页以 B 状态图为统一锚点，通过六段角色视频、自动表演和滚动章节组成一套连续的仙境体验。

## 当前交互

- 自动表演按照 `主待机 → 玩钥匙待机 → 随机角色动作 → 主待机` 循环运行。
- 点击首屏空白区域或角色画面会触发“扔钥匙”；导航、文字操作区和按钮不会误触发。
- 首页四个动画按钮分别播放扔钥匙、消失、开门和靠近；`PLAY MAD` 会随机选择动作。
- 第一次向下滚动或上滑会播放完整开门动画。只有视频触发 `ended` 后，页面才会滚动到角色档案。
- 离开首页后再次返回，会触发“靠近”回应。
- 点击 `TALK TO NYXIE` 会先播放靠近动画，再打开对话面板。

## 视频切换

页面使用两个 `<video>` 层交替播放：当前视频播放时，备用视频会预加载下一段素材。

所有动画共用一套失焦锚点转场：

1. 旧视频继续播放，整个视频舞台快速失焦。
2. 模糊达到峰值时隐藏旧视频，短暂露出 B 锚点图。
3. 新视频从 `0s` 启动，随后重新对焦。

这种方式不会叠加两个人物画面，也不使用横向扫描、烟雾或闪白遮罩。开门滚页会额外锁定当前动画，避免导演提前切回待机。

## 动画素材

| 文件 | 用途 |
| --- | --- |
| `anchor-b.webp` | 首页初始画面、视频加载回退和转场锚点 |
| `idle-main.mp4` | 主待机表演 |
| `idle-key.mp4` | 玩钥匙待机表演 |
| `react-key.mp4` | 向观察者扔钥匙 |
| `vanish.mp4` | 柴郡猫消失 |
| `portal.mp4` | 打开仙境之门 |
| `tease.mp4` | 靠近观察者 |

网页使用的压缩衍生素材位于 `assets/media/`。原始生成视频、参考视频和本地素材目录由 `.gitignore` 排除，不提交到公开仓库。

## 本地运行

项目没有第三方运行依赖，也不需要执行 `npm install`。准备好 Node.js 后运行：

```powershell
npm run dev
```

然后打开 <http://127.0.0.1:4173>。

如需更换端口：

```powershell
$env:PORT = 8080
npm run dev
```

本地服务器支持 HTTP Range 请求，浏览器可以分段读取 MP4，而不必等待整个视频下载完成。

## 项目结构

```text
Nyxie-Web/
├─ assets/media/   # 网页使用的图片和视频
├─ index.html      # 页面结构与文案
├─ styles.css      # 视觉、布局和响应式样式
├─ app.js          # 动画导演、互动和滚动控制
├─ server.mjs      # 支持 Range 的本地静态服务器
└─ package.json    # 本地运行命令
```

## 显示与兼容

- 视频默认静音自动播放，可通过右上角声音按钮切换。
- 在触摸设备上关闭磁吸、倾斜和自定义光标效果。
- 当系统启用“减少动态效果”时，首页保留静态锚点图，不启动自动视频表演。
- 页面包含桌面端和移动端响应式布局。
